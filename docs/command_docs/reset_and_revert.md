# git reset

## コマンドの概要
- **git reset**: コミット履歴を変更したり、ステージングエリアや作業ディレクトリの状態をリセットします。

## コマンド例
```bash
# ステージングエリアのみリセット
git reset --soft HEAD^

# ステージングエリアと作業ディレクトリをリセット
git reset --hard HEAD^
```

## オプションコマンド
- `--soft`: コミットを取り消し、変更をステージングエリアに残します。
- `--mixed` (デフォルト): コミットを取り消し、変更を作業ディレクトリに残します。
- `--hard`: コミット、ステージングエリア、作業ディレクトリの変更をすべて取り消します。

## 利用するケース
- 最新のコミットを取り消したい場合。
- ステージングエリアや作業ディレクトリを特定の状態に戻したい場合。

# git revert

## コマンドの概要
- **git revert**: 指定したコミットを取り消す新しいコミットを作成します。

## コマンド例
```bash
# 最新のコミットを取り消す
git revert HEAD

# 特定のコミットを取り消す
git revert <commit-hash>
```

## オプションコマンド
- `--no-commit`: 取り消しを実行するが、自動でコミットしない。
- `--edit`: 取り消しコミットのメッセージを編集可能にする。

## 利用するケース
- 過去の変更を取り消したいが、履歴を残したい場合。
- チームで共有しているリポジトリで安全に変更を取り消したい場合。
